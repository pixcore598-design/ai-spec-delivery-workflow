### 中文

我把自己在 AI 编码协作里比较稳定的一套方法，整理成了一个开源仓库：

`AI Spec Delivery Workflow`

它解决的不是“怎么让 AI 多写一点代码”，而是“怎么让 AI 更像一个可交付的工程搭档”。

核心思路很简单：

- 写代码前先对齐目标和边界
- 执行时按小步拆解推进
- 完成时必须给出测试、截图、QA 或运行证据
- 结尾明确说明改了什么、验证了什么、还有什么风险

仓库里已经包含：

- Trae 可复用 Skill
- Claude Rule
- 全局或项目级 `CLAUDE.md` 模板
- 一键安装脚本

适合 Claude Code、Trae、Cursor 这类 AI 编码工具用户直接拿去用。

仓库地址：

`https://github.com/pixcore598-design/ai-spec-delivery-workflow`

## Comment Reply Templates

Use these when people ask what the repo does.

### "What is this?"

It is a reusable workflow for AI coding agents that makes them align scope first, execute in small steps, and verify with evidence before claiming completion.

### "Why not just write a better prompt?"

Because prompts alone do not create process discipline. This repo turns good behavior into reusable defaults: spec first, execution second, verification last.

### "Who should use it?"

Anyone using Claude Code, Trae, Cursor, or similar coding agents for medium or large tasks, especially where regressions and unclear assumptions are expensive.
